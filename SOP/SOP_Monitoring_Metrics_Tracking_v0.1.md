# SOP – Metrics Tracking (v0.1)

# Purpose
Track daily progress and quality metrics for annotation work.

# Scope
In: Daily monitoring 

# Owner Role
Project Operator

# Inputs Needed
- Daily production report
- QA report

# Outputs Produced
- Metrics summary
- Trend visibility

# Tools Touched
- GitHub
- Spreadsheet (optional)

# Timebox Range
5–10 minutes daily

# Failure Modes
- Metrics not updated
- Incorrect numbers
- Trends ignored

# Escalation Rules
Escalate if throughput or quality drops suddenly

# Assumption Ledger
- Reports are filled honestly

# Step-by-Step Procedure
1. Collect daily production reports  
   Expected result: Data available

2. Collect QA reports  
   Expected result: Quality data available

3. Update metrics table  
   Expected result: Current status visible

4. Review trends  
   Expected result: Risks identified

# Embedded Quality Checks
- Numbers match reports
- Dates updated daily

# Artifacts Produced
- TPL_QA_Metrics_v0.1.md

# Example Run
Tracked daily throughput and error rate

# Failure Case
Missed updates → blind to problems
