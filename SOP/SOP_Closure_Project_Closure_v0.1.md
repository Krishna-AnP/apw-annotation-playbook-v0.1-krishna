# SOP – Project Closure (v0.1)

# Purpose
Close the annotation project cleanly after delivery.

# Scope
In: Post-delivery steps  

# Owner Role
Project Operator

# Inputs Needed
- Delivery confirmation
- Acceptance signoff

# Outputs Produced
- Closed project
- Archived documentation

# Tools Touched
- GitHub
- CVAT Cloud

# Timebox Range
10–20 minutes

# Failure Modes
- Project left open
- No documentation archive
- No lessons recorded

# Escalation Rules
Escalate if signoff not received

# Assumption Ledger
- Delivery accepted

# Step-by-Step Procedure

1. Confirm delivery acceptance  
   Expected result: Client approval received

2. Archive project artifacts  
   Expected result: No active changes

3. Revoke access if needed  
   Expected result: Security maintained

4. Mark project as closed  
   Expected result: No further work expected

# Embedded Quality Checks
- Signoff received
- Artifacts archived

# Artifacts Produced
- TPL_Acceptance_Signoff_v0.1.md

# Example Run
Toy project closed successfully

# Failure Case
No signoff → follow up before closure
